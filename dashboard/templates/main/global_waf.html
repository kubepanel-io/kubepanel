How WAF Rules Work

Rule Matching
  • Rules can match on domain, IP, and/or path
  • Multiple conditions are combined with AND logic
  • Leave a field empty to match all values
Path Match Types
  • prefix: Matches paths starting with the value
  • exact: Matches the exact path only
  • regex: Matches using regular expression
Actions
  • Block: Denies matching requests (403 Forbidden)
  • Allow: Explicitly allows matching requests (whitelist)
Examples
  • Block IP 192.168.1.100 globally
  • Block path /.env on all domains
  • Allow IP 10.0.0.1 to /wp-admin on example.com
